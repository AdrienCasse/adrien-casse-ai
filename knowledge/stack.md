# Ce que je sais faire

## Ce que j'utilise vraiment au quotidien
- Python — pandas, numpy, statsmodels, scikit-learn, FastAPI. C'est ma langue principale.
- SQL — requêtes complexes, structuration de bases, BigQuery en production.
- GCP — Cloud Run (déploiement de services Python), Cloud Functions (orchestration), BigQuery. En production réelle chez C-Ways, pas dans un tutoriel.
- Git — versionning quotidien, je sais lire un diff et écrire des commits qui ont du sens.

## Ce que je sais utiliser
- XGBoost, régression (OLS, polynomiale) — j'ai des projets concrets avec résultats chiffrés.
- R — formation académique solide, économétrie, arbres de décision, FFNN.
- Playwright, BeautifulSoup, httpx — scraping web en production.
- Next.js, React — j'ai construit des dashboards complets.
- Pydantic, aiosqlite — architecture API propre.

## Ce que je ne maîtrise pas encore
Je ne suis pas ingénieur ML en production. Je n'ai pas déployé de systèmes de deep learning à l'échelle, pas de systèmes de recommandation en production, pas de MLOps complexe. Je connais les concepts, mais je n'aurais pas l'honnêteté de les mettre en avant si je n'ai pas le recul pratique.

C'est une limite que j'assume et que je travaille activement à réduire.

## Ce qui me distingue à mon niveau
La combinaison GCP en production (Cloud Run + BigQuery) + background économétrique solide + secteur automobile/financement est rare pour un profil junior en France. La plupart des gens qui "font de la data" n'ont jamais déployé un pipeline serverless en production.
