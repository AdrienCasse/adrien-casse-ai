# Ce qu'Adrien sait faire

## Ce qu'il utilise vraiment au quotidien
- Python — pandas, numpy, statsmodels, scikit-learn, FastAPI. C'est sa langue principale.
- SQL — requêtes complexes, structuration de bases, BigQuery en production.
- GCP — Cloud Run (déploiement de services Python), Cloud Functions (orchestration), BigQuery. En production réelle chez C-Ways, pas dans un tutoriel.
- Git — versionning quotidien.
- Économétrie & statistiques — modèles de survie (Cox, Kaplan-Meier), régression OLS/polynomiale, analyse de données longitudinales. C'est son fond académique fort.

## Ce qu'il sait utiliser
- XGBoost — projets personnels avec résultats chiffrés (MAPE 4.2% sur valeur résiduelle véhicules).
- R — formation académique solide, économétrie, arbres de décision, FFNN.
- Playwright, BeautifulSoup, httpx — scraping web en production.
- Next.js, React — il a construit des dashboards complets.
- Pydantic, aiosqlite — architecture API propre.

## Ce qu'il ne maîtrise pas encore
Adrien n'est pas ingénieur ML en production au sens deep learning. Il n'a pas déployé de systèmes de recommandation à l'échelle, pas de MLOps complexe. Il connaît les concepts, mais n'aurait pas l'honnêteté de les mettre en avant sans le recul pratique.

## Ce qui distingue Adrien à son niveau
La combinaison GCP en production (Cloud Run + BigQuery) + background économétrique solide (analyses de survie, modélisation longitudinale) + secteur automobile/financement/SIV est rare pour un profil junior en France. La plupart des data scientists juniors n'ont jamais touché à des données d'immatriculation réelles ni déployé un pipeline serverless en production.
